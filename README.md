# 九寨沟景区客流动态预测系统

基于深度学习和机器学习方法，预测九寨沟景区未来 3 天客流，判断是否超过承载阈值，通过 Web 仪表盘为游客提供出行建议、为景区提供运营参考。

**核心价值**：不是精确预测人数，而是回答"未来三天适不适合去"——预测客流是否超阈值（拥挤预警）。天气信息作为辅助参考展示，不单独触发预警。

三个模型代表三类方法：**XGBoost**（传统ML）→ **GRU**（循环神经网络）→ **Transformer**（注意力机制）

> **最新更新（2026-04-20）**：精简至三模型（XGBoost/GRU/Transformer）；预测窗口 3 天滚动推理；训练数据使用 2023-06-01 起近三年记录以规避疫情/地震异常期；旺淡季样本加权；**预警逻辑改为客流主导**（超阈值即预警，天气仅作辅助说明）。

## 3. 数据子系统

- **客流爬取**：`realtime/jiuzhaigou_crawler.py`，从九寨沟官网解析每日入园人数
- **天气获取**：Open-Meteo 历史存档 API + 未来预报，自动集成于追加流程
- **数据追加**：`--append` 自动检测最后真实数据日期，批量补追空挡

**原始数据范围**：2016-01-01 ~ 至今，日级记录  
**训练使用范围**：2023-06-01 起，约 1,016 天

### 数据异常期说明

| 时期 | 原因 | 均值 | 处理方式 |
|------|------|------|---------|
| 2017-08 ~ 2018-03 | 九寨沟地震，景区闭园 | ~1,965 | 排除训练集 |
| 2020 ~ 2022 | 新冠疫情，限流/多次清零 | ~5,000 | 排除训练集 |
| 2023-06-01 起 | 正常运营恢复 | ~16,900 | 训练集起点 |

异常期识别与剔除是本项目的核心数据处理决策——训练集均值从全量数据的 9,044 提升至 16,900，与测试集分布对齐，MAE 降低约 40%。

### 关于历史数据拼接的可行性

2016-2018 年（地震前正常运营期）的季节性规律与 2023+ 基本一致，理论上可拼入训练集以增加样本量。主要风险是两段数据的绝对客流量级不同（景区扩建、政策变化），需做量级归一化对齐。当前 3 年数据已足够训练，主要瓶颈在模型结构而非数据量，暂不纳入。

## 4. 特征工程

### GRU / Transformer（11特征）

| # | 特征名 | 说明 |
|---|--------|------|
| 1 | `visitor_count_scaled` | 归一化历史客流（MinMax，训练集拟合） |
| 2 | `month_norm` | 归一化月份 `(month-1)/11` |
| 3 | `day_of_week_norm` | 归一化星期 `weekday/6` |
| 4 | `is_holiday` | 中国法定节假日标记 |
| 5 | `is_peak_season` | 旺季标记（4-11月=1） |
| 6 | `tourism_num_lag_7_scaled` | 滞后 7 天客流 |
| 7 | `tourism_num_lag_14_scaled` | 滞后 14 天客流 |
| 8 | `tourism_num_rolling_mean_14_scaled` | 14 天滚动均值 |
| 9 | `meteo_precip_sum_scaled` | 降水量 |
| 10 | `temp_high_scaled` | 最高温度 |
| 11 | `temp_low_scaled` | 最低温度 |

### XGBoost（14特征）

在 11 特征基础上额外加入 `lag_1_scaled`、`lag_28_scaled`、`rolling_std_7_scaled`（无序列窗口，需显式构造更多时序依赖）。

### 消融实验结果（GRU，2026-04-06）

| 去掉的特征 | MAE 变化 |
|-----------|---------|
| lag_7 | +10.2% |
| 天气特征 | +3.4% |
| is_holiday | +1.8% |
| 完整11特征（对照） | 最优 |

## 5. 模型架构

**推理策略**：单步训练（输出 t+1），推理时滚动3次产生 day+1/+2/+3，将前一步预测值填回输入窗口。

### GRU（主力模型）

```
输入 (45, 11) → GRU(128) → Dropout(0.1) → GRU(96) → Dropout(0.1) → Dense(32) → Dense(1)
滚动推理3次 → [day+1, day+2, day+3]
```

- 训练：MSE Loss，Adam(lr=1e-3)，EarlyStopping(patience=20)
- 样本加权：旺季(4-11月) ×2.0，淡季 ×1.0

### Transformer（Encoder-only）

```
输入 (45, 11) → PositionalEncoding → 2×EncoderBlock(d_model=64, heads=4, dff=128)
             → GlobalAveragePooling → Dense(32) → Dense(1)
滚动推理3次 → [day+1, day+2, day+3]
```

- 样本加权：旺季 ×2.0，淡季 ×1.0

### XGBoost（传统ML基线）

```
输入 14维表格特征 → XGBRegressor → 输出 t+1
滚动推理3次（lag特征手动更新）→ [day+1, day+2, day+3]
```

- 超参：`n_estimators=500, max_depth=5, learning_rate=0.03`
- 样本加权：旺季 ×2.0，淡季 ×1.0

### 集成预测（Future Work）

三模型加权平均（XGBoost×0.5 + GRU×0.3 + Transformer×0.2）预计可将整体 MAE 从 2,493 降至 2,200 左右，无需重训，仅在推理层合并。

## 6. 评估体系

**数据划分**：时序 80% / 10% / 10%，测试集为最后约 100 天  
**训练起点**：2023-06-01

### 预警判定逻辑（客流主导）

```
crowd_alert = 预测客流 ≥ 阈值（旺季 32,800 / 淡季 18,400）

risk_level:
  2 = 预警  → crowd_alert 触发（建议错峰出行）
  1 = 关注  → 仅天气异常（高温>28°C / 低温<2°C / 强降水>10mm）
  0 = 正常  → 均未触发

suitability_warning = crowd_alert（天气不单独触发预警，仅作辅助说明）
```

**设计原则**：客流超阈值是核心预警信号，权重占 80%；天气异常仅作辅助展示，不单独触发预警，避免因天气误报导致游客决策混乱。

### 回归指标

| 指标 | 定义 | 说明 |
|------|------|------|
| MAE | 平均绝对误差（人次/天） | 按 day+1/+2/+3 分别报告 |
| NRMSE | `RMSE / (y_max − y_min)` | 值域归一化，消除量纲 |
| Suit-Recall | 客流超阈值日正确识别率 | 核心指标，目标 ≥ 0.80 |

### 峰值预警评估（测试集实测）

测试集（2026-01-07 ~ 2026-04-16）中真实超淡季阈值（18,400）共 14 天：

| 时段 | 真实超阈值天数 | GRU 预测正确 | 召回率 |
|------|-------------|------------|--------|
| 2月元宵连假（2-17 ~ 2-25） | 9天 | 8天 | **89%** |
| 清明节（4-3 ~ 4-5） | 3天 | 1天 | **33%** |

清明节召回率低的原因：客流从普通日 ~8,000 一天内跳升至 38,927，模型缺少"距节假日天数"信号，无法提前感知阶跃式峰值。连续假期（元宵）因前几天客流已升高，模型能从 lag 特征感知趋势，预警效果好。

## 7. 最新性能指标

### 2026-04-18（近三年数据，80/10/10，旺淡季样本加权，单步滚动推理）

**推理策略**：单步训练（Dense(1)），推理时滚动3次产生 day+1/+2/+3 预测，无累积误差训练干扰。

| 模型 | day+1 MAE | day+2 MAE | day+3 MAE | 整体 MAE | NRMSE | Suit-Recall |
|------|-----------|-----------|-----------|---------|-------|-------------|
| **XGBoost** | 2,528 | 2,469 | 2,483 | **2,493** | 12.1% | 0.963 |
| **GRU** | 2,903 | 2,896 | 2,887 | **2,895** | 13.0% | 0.960 |
| Transformer | 3,127 | 3,103 | 3,089 | **3,106** | 13.5% | 0.960 |

**关键发现**：
- day+1 ~ day+3 误差基本持平，单步滚动推理在3步窗口内无明显累积误差
- XGBoost 特征工程充分时与深度学习模型竞争力相当
- GRU 为最稳定的深度学习模型，三天误差率约 13%
- 数据本身相邻两日客流波动均值 ~2,185 人次，模型 MAE 与自然波动处于同一量级

### 指标局限性说明

- **MAE 绝对值**：日客流值域宽（淡季 ~2,000 至旺季 ~41,000），旺季相对误差约 13%，属旅游预测领域正常水平（文献 MAPE 通常 10-20%）
- **crowd_alert 召回率**：测试集整体受限于冬季淡季；元宵连假峰值召回 89%，清明阶跃峰值召回 33%（缺少节假日距离特征，无法预感知一日暴涨）
- **早停 40-63 epoch**：样本加权导致训练/验证 loss 量纲不一致，不影响最终预测质量
